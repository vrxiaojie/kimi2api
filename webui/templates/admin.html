快速说明

账号接入方式

JWT 适合已有 token 的场景，OAuth 适合直接浏览器登录获取。

JWT 添加： 直接粘贴 access_token 或 refresh token，保存前会自动校验。

OAuth 添加： 点击启动登录后，后端会打开浏览器窗口，监听 Kimi 页面请求和存储数据，拿到有效 token 后自动保存账号。

配置位置： 项目根目录的 config.json。

接入建议

什么时候用 JWT

适用于你已经有 token，或者想快速切换多个已知账号的场景。

1. 从浏览器开发者工具或请求头中拿到 access_token / refresh token。

2. 在这里保存账号后，系统会立即访问 Kimi 的订阅接口校验有效性。

3. 如果你不想手动取 token，直接去左侧的 OAuth 登录页即可。
