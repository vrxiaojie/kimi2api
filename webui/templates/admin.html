统一管理 Kimi2API 的运行配置和账号。

左侧侧边栏会把 CLI 里的功能拆成不同视图。账号页支持手动粘贴 token，也支持直接粘贴浏览器复制的 curl 请求并提取 auth token。

快速说明

账号接入方式

支持直接粘贴 token，也支持从浏览器开发者工具复制 curl 后自动提取 auth。

手动添加： 直接粘贴 access_token、refresh token 或其他可用 token，保存前会自动校验。

curl 导入： 在浏览器 F12 的网络面板里复制任意 Kimi 请求为 curl，粘贴后后端会自动提取 Cookie 里的 auth 值并保存账号。

配置位置： 项目根目录的 config.json。

curl 导入

粘贴浏览器复制的 curl

适合你不想手动找 token，只想从开发者工具复制一次请求后直接导入。

1. 打开 https://www.kimi.com 并登录。

2. 按 F12，进入“网络”，选中任意一个发往 kimi.com 的请求，右键“复制为 curl”。

3. 粘贴到这里后，后端会从文本里提取 auth= 到第一个分号之间 的内容并立即校验。
